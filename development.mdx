<Info>
  **Prerequisites**
  - Node.js version 19 or higher
  - A docs repository containing a `docs.json` file
</Info>

## Install the CLI

Install the Mintlify CLI globally:

```bash
npm i -g mint
```

## Run the local preview

Navigate to the root of your docs repository — where your `docs.json` lives — and start the dev server:

```bash
mint dev
```

The preview opens at `http://localhost:3000` and hot-reloads as you edit files.

## Custom ports

If port 3000 is already in use, Mintlify automatically tries the next available port and logs a message:

```
Port 3000 is already in use. Trying 3001 instead.
```

You can also set a port explicitly with the `--port` flag:

```bash
mint dev --port 3333
```

## Update the CLI

Each CLI release targets a specific version of Mintlify. If your local preview looks different from production, update the CLI:

```bash
npm mint update
```

<Note>Check the [CLI changelog](https://www.npmjs.com/package/mintlify?activeTab=versions) to see what changed between versions.</Note>

## Validate links

Run the link checker before pushing to catch broken internal links:

```bash
mint broken-links
```

The command reports every broken link with its source file and line number.

For a better editing experience, install these VSCode extensions:

- [MDX](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) — syntax highlighting for `.mdx` files
- [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) — automatic code formatting on save

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>
    This error usually means your Node.js version is too old. Fix it by reinstalling the CLI against a supported Node version:

    1. Remove the current CLI: `npm remove -g mint`
    2. Upgrade Node.js to v19 or higher.
    3. Reinstall the CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Unknown error when running mint dev">
    A corrupted local cache can cause unexpected errors. Clear it and restart:

    1. Delete the cache directory: `rm -rf ~/.mintlify`
    2. Run `mint dev` again.

    <Warning>This only removes cached CLI data — your docs content is not affected.</Warning>
  </Accordion>

  <Accordion title="Local preview doesn't match the production site">
    Your CLI version may be out of sync with the deployed Mintlify version. Run `npm mint update` to pull the latest CLI, then restart the dev server with `mint dev`.
  </Accordion>
</AccordionGroup>
