---
title: 'Global settings'
description: 'Configure your documentation site by editing docs.json. Control branding, navigation, colors, and more from a single file.'
icon: 'gear'
---

Every Mintlify site requires a `docs.json` file at the root of your project. This file controls everything from your site name and colors to navigation structure and footer links.

<Tip>
Run `mint dev` after editing `docs.json` to preview changes locally at `localhost:3000`.
</Tip>

## Minimal example

<CodeGroup>

```json docs.json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "mint",
  "name": "My Docs",
  "colors": {
    "primary": "#3B82F6"
  },
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "pages": ["index", "quickstart"]
      }
    ]
  }
}
```

</CodeGroup>

## Properties

<ResponseField name="name" type="string" required>
  The name of your project. Mintlify uses this as the global site title in the browser tab and metadata.

  Example: `"KarChunT Docs"`
</ResponseField>

<ResponseField name="theme" type="string" required>
  The visual theme for your site. Choose one that matches the tone of your documentation.

  <Expandable title="Available themes">
    | Theme | Character |
    |---|---|
    | `mint` | Classic, time-tested |
    | `maple` | Modern, clean, good for AI/SaaS |
    | `palm` | Sophisticated, fintech-focused |
    | `willow` | Stripped-back, minimal |
    | `linden` | Retro terminal, monospace |
    | `almond` | Card-based, minimalist |
    | `aspen` | Modern, supports complex navigation |
    | `sequoia` | Minimal, elegant, large-scale content |
  </Expandable>
</ResponseField>

<ResponseField name="colors" type="Colors" required>
  Hex color codes for your site theme. All values must start with `#`.

  <Expandable title="Color properties">
    <ResponseField name="primary" type="string" required>
      The main accent color. Used for highlighted content, section headers, and accents in light mode.

      Example: `"#3B82F6"`
    </ResponseField>

    <ResponseField name="light" type="string">
      The accent color for dark mode. Used for highlighted content and section headers when dark mode is active.

      Example: `"#93C5FD"`
    </ResponseField>

    <ResponseField name="dark" type="string">
      The color used for important buttons and hover states.

      Example: `"#1D4ED8"`
    </ResponseField>

    <ResponseField name="background" type="object">
      Background colors for light and dark mode.

      <Expandable title="Background properties">
        <ResponseField name="light" type="string" required>
          Background hex color in light mode. Example: `"#FFFFFF"`
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Background hex color in dark mode. Example: `"#0F172A"`
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json docs.json
  "colors": {
    "primary": "#3B82F6",
    "light": "#93C5FD",
    "dark": "#1D4ED8",
    "background": {
      "light": "#FFFFFF",
      "dark": "#0F172A"
    }
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="logo" type="string | object">
  Path to your logo image, or an object with separate paths for light and dark mode. Mintlify displays the logo in the top-left corner of the navbar.

  <Expandable title="Logo properties">
    <ResponseField name="light" type="string">
      Path to the logo displayed in light mode. Example: `"/logo/light.svg"`
    </ResponseField>
    <ResponseField name="dark" type="string">
      Path to the logo displayed in dark mode. Example: `"/logo/dark.svg"`
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      The URL the logo links to when clicked. Defaults to the root of your site.
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json Single logo
  "logo": "/logo/logo.svg"
  ```

  ```json Light and dark variants
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg",
    "href": "https://example.com"
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="favicon" type="string | object">
  Path to your favicon file. You can provide a single file or separate variants for light and dark mode.

  <CodeGroup>

  ```json Single favicon
  "favicon": "/favicon.ico"
  ```

  ```json Light and dark variants
  "favicon": {
    "light": "/favicon.png",
    "dark": "/favicon-dark.png"
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="navigation" type="Navigation" required>
  Defines the sidebar navigation structure. See [Navigation](/essentials/navigation) for a full breakdown of tabs, groups, and pages.

  <Expandable title="Navigation properties">
    <ResponseField name="groups" type="Group[]">
      An array of navigation groups. Each group has a title and a list of page paths.
    </ResponseField>
    <ResponseField name="tabs" type="Tab[]">
      An array of top-level tabs. Each tab contains groups or pages.
    </ResponseField>
    <ResponseField name="anchors" type="Anchor[]">
      Persistent sidebar links visible across all sections.
    </ResponseField>
    <ResponseField name="global" type="object">
      Global anchors that appear on every page regardless of the active section.
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json docs.json
  "navigation": {
    "groups": [
      {
        "group": "Getting started",
        "pages": ["index", "quickstart"]
      },
      {
        "group": "Essentials",
        "pages": ["essentials/settings", "essentials/navigation"]
      }
    ]
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="navbar" type="Navbar">
  Configuration for the top navigation bar, including links and a primary call-to-action button.

  <Expandable title="Navbar properties">
    <ResponseField name="links" type="NavbarLink[]">
      An array of links shown in the navbar. Omit `type` for a standard text link, use `"github"` to show a repository with star count, or `"discord"` to show a server with online count.

      <Expandable title="Link properties">
        <ResponseField name="label" type="string">
          Display text for the link.
        </ResponseField>
        <ResponseField name="href" type="string">
          URL the link navigates to.
        </ResponseField>
        <ResponseField name="type" type="string">
          Link style. One of: `"github"`, `"discord"`. Omit for a standard text link.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="primary" type="object">
      The primary call-to-action button. Displayed prominently in the navbar.

      <Expandable title="Primary button properties">
        <ResponseField name="type" type="string">
          Button style. One of: `"button"`, `"github"`, `"discord"`.
        </ResponseField>
        <ResponseField name="label" type="string">
          Button label text.
        </ResponseField>
        <ResponseField name="href" type="string">
          URL the button links to.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json docs.json
  "navbar": {
    "links": [
      {
        "label": "Community",
        "href": "https://example.com/community"
      },
      {
        "type": "github",
        "href": "https://github.com/example/repo"
      }
    ],
    "primary": {
      "type": "button",
      "label": "Get started",
      "href": "https://example.com/signup"
    }
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="footer" type="Footer">
  Configuration for the site footer, including social media links and grouped resource links.

  <Expandable title="Footer properties">
    <ResponseField name="socials" type="object">
      Key-value pairs mapping social platforms to their URLs. Valid keys: `x`, `website`, `facebook`, `youtube`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`, `medium`, `telegram`, `bluesky`, `threads`, `reddit`, `podcast`.
    </ResponseField>
    <ResponseField name="links" type="FooterLinkGroup[]">
      Grouped footer links. Each group has a `header` and an array of `items` with `label` and `href`.
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json docs.json
  "footer": {
    "socials": {
      "github": "https://github.com/example",
      "x": "https://x.com/example",
      "linkedin": "https://linkedin.com/company/example"
    },
    "links": [
      {
        "header": "Resources",
        "items": [
          { "label": "Blog", "href": "https://example.com/blog" },
          { "label": "Changelog", "href": "/changelog" }
        ]
      }
    ]
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="appearance" type="Appearance">
  Controls the default color mode and whether users can toggle between light and dark.

  <Expandable title="Appearance properties">
    <ResponseField name="default" type="string" default="system">
      The default color mode for new visitors. One of: `"system"`, `"light"`, `"dark"`.
    </ResponseField>
    <ResponseField name="strict" type="boolean" default="false">
      Set to `true` to hide the light/dark mode toggle, locking users into the `default` mode.
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json Force dark mode
  "appearance": {
    "default": "dark",
    "strict": true
  }
  ```

  ```json Follow system preference
  "appearance": {
    "default": "system"
  }
  ```

  </CodeGroup>
</ResponseField>

<ResponseField name="contextual" type="Contextual">
  Configures the contextual action menu that appears when users select text on a page. This lets readers quickly copy content or open it in an AI assistant.

  <Expandable title="Contextual properties">
    <ResponseField name="options" type="string[]">
      An array of options to show in the menu. Available values: `"copy"`, `"view"`, `"chatgpt"`, `"claude"`, `"perplexity"`, `"mcp"`, `"cursor"`, `"vscode"`.
    </ResponseField>
  </Expandable>

  <CodeGroup>

  ```json docs.json
  "contextual": {
    "options": ["copy", "claude", "cursor", "chatgpt"]
  }
  ```

  </CodeGroup>

  <Note>
  Users see only the options you include. Keep the list short to avoid overwhelming readers.
  </Note>
</ResponseField>

## Full example

The following shows a complete `docs.json` configuration combining all major settings:

<CodeGroup>

```json docs.json
{
  "$schema": "https://mintlify.com/docs.json",
  "theme": "maple",
  "name": "KarChunT Docs",
  "colors": {
    "primary": "#6366F1",
    "light": "#A5B4FC",
    "dark": "#4338CA"
  },
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg",
    "href": "/"
  },
  "favicon": "/favicon.ico",
  "navbar": {
    "links": [
      { "type": "github", "href": "https://github.com/example/repo" }
    ],
    "primary": {
      "type": "button",
      "label": "Get started",
      "href": "/quickstart"
    }
  },
  "footer": {
    "socials": {
      "github": "https://github.com/example",
      "x": "https://x.com/example"
    }
  },
  "appearance": {
    "default": "system"
  },
  "contextual": {
    "options": ["copy", "claude", "cursor"]
  },
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Getting started",
            "pages": ["index", "quickstart"]
          },
          {
            "group": "Essentials",
            "pages": ["essentials/settings", "essentials/navigation"]
          }
        ]
      }
    ]
  }
}
```

</CodeGroup>
