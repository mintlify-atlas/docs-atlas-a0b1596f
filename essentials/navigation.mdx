Mintlify builds your sidebar from the `navigation` object in `docs.json`. Pages do not appear in the sidebar automatically — you must add each page to the navigation structure.

<Note>
You will edit `docs.json` every time you add a new page. New MDX files that are not listed in `navigation` are accessible by URL and through search, but they will not appear in the sidebar.
</Note>

## How file paths map to URLs

Page paths in `docs.json` correspond directly to file paths relative to your project root. Omit the `.mdx` extension.

| File path | Page path in docs.json | URL |
|---|---|---|
| `index.mdx` | `"index"` | `/` |
| `quickstart.mdx` | `"quickstart"` | `/quickstart` |
| `guides/setup.mdx` | `"guides/setup"` | `/guides/setup` |
| `api/reference/users.mdx` | `"api/reference/users"` | `/api/reference/users` |

<Warning>
Do not use `api` as a top-level folder name. Mintlify runs on Next.js, which reserves the top-level `api` folder for internal server routes. Use a name like `api-reference` instead.
</Warning>

## Groups

Groups are the basic unit of sidebar organization. Each group has a title and an array of page paths.

<CodeGroup>

```json docs.json
"navigation": {
  "groups": [
    {
      "group": "Getting started",
      "pages": ["index", "quickstart"]
    },
    {
      "group": "Essentials",
      "pages": [
        "essentials/settings",
        "essentials/navigation"
      ]
    }
  ]
}
```

</CodeGroup>

### Group properties

<ResponseField name="group" type="string" required>
  The section title displayed in the sidebar.
</ResponseField>

<ResponseField name="pages" type="string[] | (string | Group)[]" required>
  An array of page paths. You can nest a group object inside `pages` to create a collapsible sub-group.
</ResponseField>

<ResponseField name="icon" type="string">
  An icon displayed next to the group title. Accepts Lucide, Font Awesome, or Tabler icon names.
</ResponseField>

<ResponseField name="tag" type="string">
  A label displayed next to the group name. Useful for marking sections as new or beta.
</ResponseField>

<ResponseField name="expanded" type="boolean">
  Controls the default open state for nested groups. Top-level groups are always expanded.
</ResponseField>

### Nested groups

You can nest a group inside a `pages` array to create a collapsible sub-section.

<CodeGroup>

```json docs.json
"navigation": {
  "groups": [
    {
      "group": "Guides",
      "pages": [
        "guides/overview",
        {
          "group": "Advanced",
          "expanded": false,
          "pages": [
            "guides/advanced/config",
            "guides/advanced/deploy"
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Tabs

Tabs add a top-level navigation bar below the navbar. Use tabs when you have distinct sections with different audiences or content types, such as documentation and an API reference.

Each tab contains `groups` or a flat `pages` array. You can also link a tab to an external URL.

<CodeGroup>

```json docs.json
"navigation": {
  "tabs": [
    {
      "tab": "Documentation",
      "icon": "book-open",
      "groups": [
        {
          "group": "Getting started",
          "pages": ["index", "quickstart"]
        },
        {
          "group": "Essentials",
          "pages": ["essentials/settings", "essentials/navigation"]
        }
      ]
    },
    {
      "tab": "API reference",
      "icon": "square-terminal",
      "pages": ["api-reference/overview", "api-reference/endpoints"]
    },
    {
      "tab": "Blog",
      "icon": "newspaper",
      "href": "https://example.com/blog"
    }
  ]
}
```

</CodeGroup>

### Tab properties

<ResponseField name="tab" type="string" required>
  The tab label displayed in the tab bar.
</ResponseField>

<ResponseField name="groups" type="Group[]">
  An array of groups shown in the sidebar when this tab is active.
</ResponseField>

<ResponseField name="pages" type="string[]">
  A flat list of pages when you do not need groups within a tab.
</ResponseField>

<ResponseField name="href" type="string">
  An external URL. The tab acts as a link instead of switching sidebar content.
</ResponseField>

<ResponseField name="icon" type="string">
  An icon displayed next to the tab label.
</ResponseField>

## Anchors

Anchors are persistent links pinned to the top of the sidebar. They remain visible regardless of which tab or section is active, making them ideal for links like a changelog, community forum, or status page.

<CodeGroup>

```json docs.json
"navigation": {
  "anchors": [
    {
      "anchor": "Changelog",
      "icon": "list",
      "groups": [
        {
          "group": "Releases",
          "pages": ["changelog/2024", "changelog/2023"]
        }
      ]
    },
    {
      "anchor": "Community",
      "icon": "discord",
      "href": "https://discord.gg/example"
    }
  ]
}
```

</CodeGroup>

### Global anchors

Global anchors appear on every page, even when the user is inside a tab or product section. Define them under `navigation.global.anchors`.

<CodeGroup>

```json docs.json
"navigation": {
  "global": {
    "anchors": [
      {
        "anchor": "Changelog",
        "icon": "list",
        "href": "/changelog"
      },
      {
        "anchor": "Support",
        "icon": "life-ring",
        "href": "https://example.com/support"
      }
    ]
  },
  "tabs": [
    {
      "tab": "Documentation",
      "groups": [
        {
          "group": "Getting started",
          "pages": ["index", "quickstart"]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Choosing a navigation pattern

Mintlify supports several navigation patterns. Use the simplest one that meets your needs.

<AccordionGroup>
  <Accordion title="Groups — default choice">
    Use groups when your site has a single audience and a straightforward hierarchy. Groups appear directly in the sidebar with no top-level tabs.

    ```json
    "navigation": {
      "groups": [
        { "group": "Getting started", "pages": ["index", "quickstart"] }
      ]
    }
    ```
  </Accordion>

  <Accordion title="Tabs — distinct content types">
    Use tabs when you have sections with different audiences or purposes, such as user guides and an API reference. Each tab has its own sidebar.

    ```json
    "navigation": {
      "tabs": [
        { "tab": "Guides", "groups": [{"group": "Start", "pages": ["index"]}] },
        { "tab": "API", "pages": ["api/overview"] }
      ]
    }
    ```
  </Accordion>

  <Accordion title="Anchors — persistent sidebar links">
    Use anchors for links that should always be visible, like a changelog or status page. Combine with `global` to keep them visible across all tabs.
  </Accordion>

  <Accordion title="Dropdowns — section switcher">
    Use dropdowns when you have multiple sections that users switch between and want a compact switcher in the navbar area.
  </Accordion>

  <Accordion title="Products — multi-product companies">
    Use products when your company ships multiple separate products, each with its own full navigation structure.
  </Accordion>
</AccordionGroup>

## Hidden pages

Any MDX file not listed in `docs.json` is treated as a hidden page. Hidden pages are:

- Excluded from the sidebar
- Accessible by direct URL
- Discoverable through the search bar

<Tip>
Use hidden pages for draft content or internal reference material you want available by direct link but not surfaced in the main navigation.
</Tip>
