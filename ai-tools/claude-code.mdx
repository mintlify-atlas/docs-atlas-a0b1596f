Claude Code is Anthropic's terminal-based AI coding agent. You can point it at your docs directory and use it to draft pages, update navigation, fix broken markup, and maintain consistency across a large documentation set.

## Prerequisites

- Active Claude subscription (Pro, Max, or API access)

## Setup

<Steps>
  <Step title="Install Claude Code">
    Install the Claude Code CLI globally:

    ```bash
    npm install -g @anthropic-ai/claude-code
    ```

    After installation, authenticate by running `claude` and following the login prompt.
  </Step>

  <Step title="Install the Mintlify skill">
    Navigate to your docs directory and run:

    ```bash
    npx skills add https://mintlify.com/docs
    ```

    This installs a skill file that Claude Code reads automatically at the start of every session. The skill provides:

    - **Component reference** — correct props and usage for every Mintlify component
    - **Writing standards** — tone, structure, and formatting conventions
    - **Workflow guidance** — how to add pages, update `docs.json`, and preview changes
  </Step>

  <Step title="Customize AGENTS.md">
    Open `AGENTS.md` in your project root and add project-specific context:

    - Preferred terminology and any terms to avoid
    - Style rules (voice, heading case, link conventions)
    - Content scope — what Claude Code should and should not modify

    Claude Code reads this file on startup, so changes take effect immediately the next time you run `claude`.
  </Step>

  <Step title="Start Claude Code">
    From your docs directory, start a session:

    ```bash
    claude
    ```

    You can now give natural-language instructions such as "Add a new page about authentication under the Guides section" or "Rewrite the introduction of `quickstart.mdx` in second person."
  </Step>
</Steps>

<Note>
  The Mintlify skill gives Claude Code accurate, up-to-date knowledge of components, configuration options, and doc structure. Without it, Claude Code may generate plausible-looking but incorrect component syntax.
</Note>
